# Policy: anti_generic_output_policy

## Goal

Prevent generic, low-signal outputs from becoming the default quality bar of Atlas.

## Requirements

- define audience before proposing UX or branding work
- define value proposition before naming or visual direction
- avoid generic defaults without rationale
- ask for references or constraints when the task depends on them
- document when a result is intentionally generic as a placeholder

## Applies to

- brand direction
- UX/UI guidance
- project positioning docs
- templates and prompts that may influence downstream outputs
