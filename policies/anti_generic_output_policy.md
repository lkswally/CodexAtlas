# Policy: anti_generic_output_policy

## Goal

Prevent generic, low-signal outputs from becoming the default quality bar of Atlas.

## Requirements

- define audience before proposing UX, branding or product positioning work
- define value proposition before naming, copy or visual direction
- avoid default SaaS-like structure, palette and typography unless there is an explicit reason
- ask for references, constraints or mood direction when the task depends on them
- document when a result is intentionally generic as a placeholder
- distinguish between a temporary scaffold and a quality-reviewed creative output
- prefer differentiated visual direction over safe sameness when the task calls for branding or UX judgment

## Applies to

- brand direction
- UX/UI guidance
- project positioning docs
- templates and prompts that may influence downstream outputs
- product-branding reviews
- bootstrap templates that could become downstream defaults
