# Visual Direction Policy

Atlas should not move from vague product desire to UI implementation without an explicit visual direction.

## Minimum requirements

- audience is explicit or clearly marked as unknown
- mood or vibe is explicit
- originality level is explicit
- the hero direction is explicit
- the review states what the project should avoid, not only what it wants

## Warning signals

- "just make it look good"
- "you decide everything"
- no audience
- no mood or vibe
- no originality target

## Atlas behavior

- prefer `visual-direction-checkpoint` before implementation when direction is vague
- treat missing direction as a quality warning, not as license to default to generic UI
- keep the output structured and explainable
