# Change Proposal Workflow

Use this workflow for medium or large Atlas changes that benefit from explicit scope, requirements, design and verification before implementation.

## 1. Proposal

Capture:

- what changes
- why now
- what problem it resolves
- what stays out of scope
- risks
- expected impact

## 2. Specs

Capture:

- functional requirements
- non-functional requirements
- scenarios
- acceptance criteria

## 3. Design

Capture:

- technical approach
- affected modules
- tradeoffs
- discarded alternatives

## 4. Tasks

Capture:

- small tasks
- suggested order
- validation per task
- rollback if needed

## 5. Verify

Capture:

- tests
- governance
- quality gate
- minimum evidence
- known false positives

## 6. Archive

Capture:

- final decision
- commit reference
- lessons learned
- next steps

## Atlas Boundaries

- This workflow is documentary and advisory.
- It does not apply changes automatically.
- It does not replace governance or quality gate.
- It should stay lightweight for small fixes and stronger for medium or large work.
